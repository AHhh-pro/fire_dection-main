## 测试内容详解

### 1. 单元测试 (test_unit.py)

针对系统最小可测试单元的验证：

- **温度数据处理单元**
  - 温度数据格式验证
  - 温度分级分类（正常/中等/高温）

- **警报逻辑单元**
  - 火灾警报级别判断
  - 警报消息生成

- **模型配置单元**
  - 模型路径构建
  - 置信度阈值验证

- **数据处理单元**
  - 检测统计聚合

- **工具函数单元**
  - 时间格式转换
  - JSON数据解析

### 2. 功能测试 (test_functional.py)

验证系统各项功能是否按需求正常工作：

- **温度数据接收功能**
  - C#数据格式兼容性
  - 温度显示更新逻辑
  - 温度超时处理

- **火灾检测功能**
  - 检测类别验证
  - 警报级别转换

- **模型加载功能**
  - 模型文件存在性检查
  - 模型选择选项

- **GUI组件功能**
  - 输入源类型
  - 置信度范围

- **HTTP通信功能**
  - POST请求格式

- **数据持久化功能**
  - 结果保存
  - 日志记录

### 3. 集成测试 (test_integration.py)

验证各模块之间的协同工作：

- **温度数据集成**
  - C#到Python的通信链路
  - 多传感器数据接收

- **检测与警报集成**
  - 火灾检测到警报的完整流程
  - 警报到UI更新的流程

- **模型与检测集成**
  - 模型到检测的完整流程
  - 检测结果格式标准化

- **GUI功能集成**
  - 输入源切换
  - 置信度到检测过滤

- **数据流集成**
  - 端到端数据流
  - 并发数据处理

### 4. 性能测试 (test_performance.py)

测试系统在各种负载下的性能表现：

- **响应时间测试**
  - 温度更新响应时间（目标<10ms）
  - 警报计算性能

- **吞吐量测试**
  - HTTP请求吞吐量（目标>50请求/秒）
  - 检测处理吞吐量（目标>20 FPS）

- **内存使用测试**
  - 温度数据内存增长
  - 检测结果内存清理

- **并发测试**
  - 并发温度更新
  - 线程安全性

- **可扩展性测试**
  - 目标数量扩展性

### 5. 异常测试 (test_exception.py)

测试系统在异常情况下的处理能力：

- **无效输入处理**
  - 无效温度值
  - 缺少传感器ID
  - 畸形JSON数据

- **网络错误处理**
  - 连接被拒绝
  - 超时处理
  - 无效URL

- **资源错误处理**
  - 模型文件缺失
  - 无效模型路径
  - 权限不足

- **数据损坏处理**
  - 损坏的温度数据
  - 空数据结构

- **边界条件测试**
  - 温度边界值
  - 置信度边界值
  - 字符串长度边界

- **恢复机制测试**
  - 优雅降级
  - 状态恢复

- **并发错误测试**
  - 竞态条件处理

## 注意事项

1. 运行集成测试前确保端口 18090 未被占用
2. 性能测试可能需要较长时间，请耐心等待
3. 部分测试需要网络连接（如超时测试）
4. 建议在干净的环境中运行测试

## 问题排查

### 测试失败常见原因

1. **模型文件不存在**
   - 确保 `weights/` 目录包含模型文件

2. **端口被占用**
   - 修改集成测试中的端口号

3. **权限不足**
   - 以管理员身份运行测试

4. **依赖缺失**
   - 安装所需依赖：`pip install -r requirements.txt`
